# Промпт для обработки транскрипций

Универсальный промпт для последовательного анализа транскрипций встреч с использованием LLM (Claude, GPT-4 и др.).

## Что это?

Структурированный промпт, который превращает сырую транскрипцию в:
- Список участников с ролями
- Исправленный текст (ошибки ASR, слова-паразиты)
- Структурированные вопросы и ответы
- Тезисы по темам
- Навигацию по таймкодам
- Список решений и задач
- Оценку зрелости процессов (опционально)

## Быстрый старт

### 1. Подготовь контекст

```yaml
тип_встречи: интервью
технический_домен: DevOps, AppSec
термины: Kubernetes, SAST, DAST, Keycloak, GitLab
```

### 2. Запусти анализ

```
Вот транскрипция встречи: [прикрепи файл]

Контекст:
- Тип: интервью по AppSec
- Домен: DevOps, AppSec

Выполни протоколы 1-7 последовательно.
Жди подтверждения после каждого.
```

### 3. Подтверждай каждый протокол

После каждого протокола отвечай "ок" или "далее" для продолжения.

## Протоколы

| № | Протокол | Что делает |
|---|----------|------------|
| 1 | Участники | Идентификация спикеров, ролей, организаций |
| 2 | Ошибки ASR | Исправление искажений распознавания речи |
| 3 | Очистка текста | Слова-паразиты, повторы, междометия |
| 4 | Вопросы и ответы | Извлечение Q&A, группировка по темам |
| 5 | Темы и тезисы | Выделение тем, ключевые факты, противоречия |
| 6 | Навигация | Оглавление по таймкодам |
| 7 | Решения и задачи | Action items, дедлайны, gaps |
| 8 | Оценка зрелости | Шкала 1-5 по процессам (опционально) |

## Режимы работы

### Детализация

```yaml
детализация: краткая      # только статистика + топ-5
детализация: стандартная  # полные таблицы (по умолчанию)
детализация: полная       # + цитаты, расширенный анализ
```

### Выборочные протоколы

```
Выполни только протоколы 2 и 7.
```

### Пакетный режим (без подтверждений)

```
Выполни все протоколы подряд без ожидания.
Выдай единый отчёт.
```

## Особенности v3.0

### Словарь технических терминов

Промпт содержит предустановленный словарь терминов DevOps/AppSec для распознавания ASR-искажений:

| ASR-искажение | Правильно |
|---------------|-----------|
| киклог | Keycloak |
| обсек | AppSec |
| Волки | Valkey |
| даст | DAST |

### Идентификация без маркеров спикеров

Если в транскрипции нет меток `[Спикер 1]:`, промпт использует:
- Прямые обращения: "Андрей, скажи пожалуйста"
- Представления: "Я — Дмитрий, РП"
- Контекст ответов

### Связь gaps → задачи

Протокол 7 показывает, какая задача закрывает какой gap:

| Gap | Задача | Ответственный |
|-----|--------|---------------|
| SAST не в pipeline | Внедрить AI | SomeCompany + Заказчик |
| Доступы не отзываются | — | — (gap не закрыт!) |

### Валидация таймкодов

- Формат: `[ММ:СС]` где СС < 60
- Если секунды ≥ 60 — используется `[строка N]`

## Выходные артефакты

После обработки получаешь:
- **Словарь автозамен** — для массовой замены в редакторе
- **Список открытых вопросов** — для follow-up
- **Задачи с ответственными** — для трекинга
- **Gaps без задач** — требуют внимания
- **Оценка зрелости** — для планирования улучшений

## Примеры использования

### Обработка технического интервью

```
Контекст:
- Тип: интервью по инфраструктуре
- Домен: DevOps, Kubernetes
- Термины: K8s, Helm, ArgoCD, Terraform

Параметры:
  детализация: стандартная
  оценка_зрелости: да

[прикрепить файл транскрипции]
```

### Быстрый анализ совещания

```
Параметры:
  детализация: краткая

Выполни протоколы 1, 5, 7 (участники, темы, задачи).
```

### Только исправление ошибок

```
Выполни только Протокол 2 (ошибки ASR).
Выдай словарь автозамен.
```

## Требования

- LLM с поддержкой длинного контекста (Claude, GPT-4)
- Транскрипция в текстовом формате
- Желательно наличие таймкодов `[мм:сс]`

## Советы

1. **Заполняй контекст** — чем больше терминов укажешь, тем лучше распознавание ASR
2. **Проверяй первые протоколы** — ошибки в идентификации участников каскадируются
3. **Используй краткий режим** для первичного просмотра длинных транскрипций
4. **Сохраняй словарь автозамен** — пригодится для обработки следующих файлов той же серии встреч
